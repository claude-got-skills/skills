## Methodology

Each test runs twice: once as **Control** (bare system prompt: "You are a helpful AI assistant…") and once as **Treatment** (system prompt includes the full SKILL.md + reference files). Both conditions use `max_tokens=4096`, no tools, direct answer only.

Scoring is dual-method:
1. **Keyword scoring** — checks for accuracy keywords (correct facts), completeness keywords (depth/detail), and deprecated pattern detection (outdated info)
2. **LLM judge** — Haiku 4.5 rates each response on accuracy (0-3), completeness (0-3), and actionability (0-3) using SKILL.md as ground truth rubric

## Key Findings

### 1. Consistent uplift across both models
The skill produces strong accuracy gains on both Haiku (+2.9 mean) and Sonnet (+2.7 mean) across positive test categories. The uplift is largest for Model Selection (+5.0 / +4.0) and Implementation Guidance (+3.3 / +3.3).

### 2. Sonnet benefits slightly more on completeness
Sonnet treatment responses are more complete than Haiku's in most categories, particularly Architecture Decisions (Sonnet +2.67 vs Haiku +2.00) and Implementation Guidance (Sonnet +3.33 vs Haiku +3.00).

### 3. No regression on general knowledge
Negative tests (Python decorators, REST API design, Git branching) show near-zero regression, confirming the skill content doesn't interfere with non-capabilities queries.

### 4. Hallucination prevention works
Both models go from near-zero accuracy on hallucination tests to 3+ with the skill, successfully avoiding confident claims about nonexistent features.

### 5. Token overhead is ~97k
The full skill content (SKILL.md + references) adds approximately 97,000 tokens to the system prompt. This is significant but justified by the accuracy gains, especially for capability-sensitive queries.
