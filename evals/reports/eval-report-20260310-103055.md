# Simple Document Analysis Pipeline

Here's a practical architecture that scales without over-engineering:

## Core Flow

```
PDF Input → Text Extraction → Chunking → LLM Analysis → 
Knowledge Base Lookup → Summary Generation → Structured Output
```

## Recommended Stack

**Processing Layer:**
- **PyPDF2** or **pdfplumber** for extraction (pdfplumber handles tables better)
- **LangChain** or **LlamaIndex** to orchestrate the pipeline
- **sentence-transformers** for embeddings if you need semantic search

**LLM & Storage:**
- Claude API (or OpenAI/local LLM) for analysis
- **Pinecone** or **Weaviate** for vector search (or PostgreSQL with pgvector if you want to self-host)
- Simple **SQLite/PostgreSQL** for structured metadata

**Async & Scaling:**
- **Celery** + **Redis** (if processing multiple PDFs concurrently)
- Or just **Python async/ThreadPoolExecutor** for a small team's workload

## Practical Implementation Sketch

```python
from langchain.document_loaders import PDFPlumberLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain.chat_models import ChatAnthropic
from langchain.chains import RetrievalQA

# 1. Load & chunk
loader = PDFPlumberLoader("document.pdf")
docs = loader.load()
splitter = RecursiveCharacterTextSplitter(chunk_size=1000, overlap=100)
chunks = splitter.split_documents(docs)

# 2. Embed & store (reuse existing embeddings)
embeddings = HuggingFaceEmbeddings(model_name="all-MiniLM-L6-v2")
vectorstore = Pinecone.from_documents(chunks, embeddings, index_name="kb")

# 3. Query with context
qa_chain = RetrievalQA.from_chain_type(
    llm=ChatAnthropic(),
    chain_type="stuff",  # or "map_reduce" for longer docs
    retriever=vectorstore.as_retriever()
)

summary = qa_chain.run("Summarize key findings and cross-reference with known risks")
```

# Document Analysis Pipeline Architecture

Given your constraints (100-page PDFs, small team, need for structured output), here's a straightforward approach:

## Recommended Architecture

**Single-pass processing with Claude + Files API + Structured Outputs**

```
PDF → Files API → Claude (1M context) → Structured JSON → Your DB
```

### Why This Works

1. **100-page PDFs fit in one pass**: A 100-page PDF is typically 50–100K tokens. With 1M context (beta on Opus 4.6/Sonnet 4.6), you have room for:
   - The PDF itself (~80K tokens)
   - Your knowledge base (~200K tokens compressed)
   - System prompt + examples (~20K tokens)
   - Output buffer
   
   This avoids chunking complexity entirely.

2. **Structured outputs guarantee JSON**: Use `output_config.format` with a schema defining your summary structure. No post-processing parsing needed.

3. **Files API for scalability**: Upload PDFs once, reference by `file_id` across requests. Handles the 32MB limit cleanly.

4. **Prompt caching for the knowledge base**: Cache your KB (5-min or 1-hour) so repeated analyses don't reprocess it.

# QA System Architecture for AI Content

Here's a practical multi-layer approach:

## 1. **Automated Checks (First Pass)**

```python
class ContentQASystem:
    def __init__(self):
        self.validators = [
            StyleGuideValidator(),
            ToneAnalyzer(),
            FactualityChecker(),
            ComplianceValidator()
        ]
    
    def review(self, content, metadata):
        results = {
            'passed': True,
            'issues': [],
            'flags': []
        }
        
        for validator in self.validators:
            check_result = validator.validate(content, metadata)
            if not check_result.passed:
                results['passed'] = False
                results['issues'].extend(check_result.issues)
            results['flags'].extend(check_result.flags)
        
        return results
```

**What to check automatically:**
- Keyword presence/absence (style guide requirements)
- Tone markers (sentiment, formality level)
- Length constraints
- Forbidden terms/patterns
- Required disclosures
- URL/reference validity (where feasible)

## 2. **Accuracy Verification**

This is harder to fully automate:

```python
class FactualityChecker:
```

```python
def route_for_review(qa_results, content_type):
    # Auto-approve if low-risk
    if qa_results['passed'] and content_type == 'general_info':
        return 'auto_approve'
    
    # Route to appropriate reviewer
    if qa_results['issues']:
        severity = max(issue['severity'] for issue in qa_results['issues'])
        if severity == 'critical':
            return 'expert_review'
        elif 'factual' in [i['type'] for i in qa_results['issues']]:

        else:
```

```typescript
  const [messages, setMessages] = useState<Message[]>([]);

          if (done) break;

          for (const line of lines) {

                  content: assistantResponse,
                };
                return newMessages;
              });
            }
          }
```

```python
from anthropic import Anthropic

client = Anthropic()

class CachedConversationManager:
    def __init__(self, summary_threshold=10):
        self.messages = []
        self.summary = None
        self.summary_threshold = summary_threshold
    
    def add_message(self, role, content):

        if len(self.messages) >= self.summary_threshold:

        if len(self.messages) < self.summary_threshold:
            return
        
        # Summarize all but the last few messages
        to_summarize = self.messages[:-3]
        recent = self.messages[-3:]
        
        response = client.messages.create(
            model="claude-3-5-sonnet-20241022",

        messages_for_api = []
```

```javascript
const http = require('http');
const { execSync } = require('child_process');
const path = require('path');

const server = http.createServer((req, res) => {
  if (req.method === 'POST' && req.url === '/check-python') {
    let body = '';
    req.on('data', chunk => body += chunk);
    req.on('end', () => {
      const { filePath } = JSON.parse(body);
```

```python
def sort_by_key(
    data: List[Dict[str, Any]],
    key: str,
    reverse: bool = False,
    default: Optional[Any] = None,
    missing_key_error: bool = False
) -> List[Dict[str, Any]]:
    """
    Sort a list of dictionaries by a specified key.
    
    Args:
        data: List of dictionaries to sort
        key: Dictionary key to sort by
        reverse: If True, sort in descending order (default: False)
        default: Default value for missing keys (default: None)
                 If None, missing keys sort to the beginning
        missing_key_error: If True, raise ValueError when key is missing
                          If False, use default value (default: False)
    
    Returns:
        Sorted list of dictionaries
    
    Raises:
        ValueError: If missing_key_error=True and key not found in any dict
        TypeError: If key is not a string
    """
    if not isinstance(key, str):
        raise TypeError(f"Key must be a string, got {type(key).__name__}")
    
    # Check if key exists in all dictionaries
    missing_keys = [i for i, d in enumerate(data) if key not in d]
    
    if missing_keys and missing_key_error:
        raise ValueError(
            f"Key '{key}' not found in dictionaries at indices: {missing_keys}"
        )
    
    # Sort with a custom key function that handles missing values
    return sorted(
        data,
        key=lambda d: (d.get(key) is None, d.get(key, default)),
```
