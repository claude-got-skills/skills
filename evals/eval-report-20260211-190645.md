# TCP vs UDP: Core Differences

## TCP (Transmission Control Protocol)
**Connection-oriented, reliable delivery**

- **Guaranteed delivery**: Packets are acknowledged and retransmitted if lost
- **Ordered**: Data arrives in the exact sequence sent
- **Error-checked**: Built-in checksums detect corruption
- **Flow control**: Adjusts transmission rate based on network conditions
- **Connection setup**: Three-way handshake before data transfer (overhead)
- **Slower**: Due to acknowledgments and potential retransmissions

## UDP (User Datagram Protocol)
**Connectionless, best-effort delivery**

- **No guarantees**: Packets may be lost, duplicated, or arrive out of order
- **No connection**: Just sends data immediately (lower latency)
- **Minimal overhead**: Smaller packet headers, no acknowledgments
- **Faster**: No waiting for confirmations or retransmissions
- **Simpler**: Less protocol complexity

- **Web traffic** (HTTP/HTTPS) — pages must load completely
- **Email** (SMTP, IMAP) — messages can't have missing data
- **File transfers** (FTP, SFTP) — files must arrive intact
- **Database connections** — queries need guaranteed responses
- **SSH/remote terminals** — every command must execute
- **APIs/REST services** — request/response integrity critical

**Key principle**: If you can't tolerate data loss, use TCP.

**Use when speed/latency matters more than perfect delivery:**

- **Live streaming** (video/audio) — skip corrupted frames, keep playing

- **DNS queries** — simple request/response, just retry if needed

- **Network time protocol (NTP)** — approximate time is fine

**Key principle**: If fresh data matters more than complete data, use UDP.

- **Rust toolchain** (rustc, cargo)
- **Common languages**: Python, Node.js, Java, C++, Go, and more
- **Isolated sandbox**: secure execution with network access disabled by default

## Basic Example (API)

```python
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4.5-20250514",
    max_tokens=4096,
    tools=[{
        "type": "code_interpreter_2024_08_29",
        "name": "code_execution"
    }],
    messages=[{
        "role": "user",
        "content": "Write and run a Rust program that calculates Fibonacci numbers"
    }]
)
```

Claude will:
1. Write Rust code (e.g., `main.rs`)
2. Compile it with `rustc` or `cargo build`
3. Execute the binary
4. Return the output

## Claude Code Example

In Claude Code (CLI or IDE), just ask:

```bash
claude "Write a Rust CLI tool that parses JSON"
```

Claude will create the project structure, write `Cargo.toml` + source files, compile, and run tests.

## Important Limitations

⚠️ **Network access is disabled** in the sandbox — you can't fetch dependencies from crates.io at runtime. For crates beyond std:

**Option 1: Use pre-installed crates**  
Check `references/agent-capabilities.md` for the list of pre-installed crates. Common ones like `serde`, `regex`, `tokio` are available.

**Option 2: Vendor dependencies**  
Provide the crate source code directly in your prompt or via files.

**Option 3: Use Claude Code with skills**  
Create a skill that sets up a full Rust environment with `cargo vendor` or pre-downloads dependencies before execution.

## When to Use Which Tool

- **Simple scripts/one-off tasks**: `code_interpreter_2024_08_29` via API
- **Full project workflows** (multi-file, dependencies, testing): Claude Code with Bash tool enabled
- **Production builds**: Use code execution for prototyping, then move to your own CI/CD

See `references/tool-types.md` (section: Code Execution) for the complete language list, pre-installed packages, and container specs.
